# Интеграция Payform

## Что сейчас работает

Система поддерживает **две способа оплаты** подписки:

1. **Payform** (основной) — ваша готовая платёжная ссылка с автоматической активацией подписки
2. **По номеру телефона** (резервный) — для постоянных клиентов, которые знают и доверяют вам

## Как это устроено

### Для новых/постоянных клиентов (Payform):

```
Клиент в ЛК → видит кнопку "Оплатить XXXX ₽" → 
нажимает → переходит на https://luchistaya-yoga.payform.ru?amount=3500 (или 3000) → 
платит картой/СБП → Payform отправляет вебхук → 
подписка активируется **автоматически** ✓
```

**Суммы:**
- Первая оплата: **3500 ₽**
- Последующие: **3000 ₽**

### Для постоянных клиентов (телефон):

```
Клиент нажимает "По номеру телефона" → 
видит номер +7 912 588-08-26 → 
платит через банк СБП → 
нажимает "Нажми после оплаты" → 
пишет вам в Telegram → 
вы вручную активируете подписку
```

## Настройка

### 1. Webhook в Payform

1. Откройте админку Payform (внутри вашего аккаунта Prodamus)
2. Перейдите в **Интеграция** → **Webhooks**
3. Добавьте вебхук:
   - **URL**: `https://ваш-домен.ru/api/webhook/prodamus.php`
   - **События**: выберите все события платежа (success, failed, cancelled)

### 2. Ссылка на вашу форму

Система использует ссылку: `https://luchistaya-yoga.payform.ru`

Убедитесь, что она работает и в неё можно передавать параметр `amount`:
```
https://luchistaya-yoga.payform.ru?amount=3500
```

### 3. Примените миграцию БД (опционально)

Если раньше вы не применяли:
```bash
mysql -u пользователь -p база < db/migrations/0006_add_prodamus_support.sql
```

## Файлы

- `cabinet/partials/payment-block.php` — блок оплаты с двумя способами
- `api/webhook/prodamus.php` — обработка вебхуков (автоактивация подписки)
- `api/payment/mark-pending.php` — отметить платёж как "в ожидании"
- `db/migrations/0006_add_prodamus_support.sql` — расширение БД

## Как проверить

1. **Заходите в ЛК под новым пользователем**
   - Должны видеть две кнопки: "Карта или СБП" и "По номеру телефона"
   - По умолчанию выбрана "Карта или СБП"
   - Сумма должна быть 3500 ₽

2. **Заходите под постоянным клиентом** (у кого есть платежи в БД)
   - Сумма должна быть 3000 ₽

3. **Тестируйте переходы**
   - "Карта или СБП" → переходит на ваш Payform с нужной суммой
   - "По номеру телефона" → показывает ваш номер и кнопку в Telegram

## Логи

Проверяйте логи оплат:
```bash
tail -f logs/error.log | grep -i payform
```

## Постоянные клиенты

Когда клиент совершает первый платёж (3500 ₽) → система его помнит как "постоянный клиент" → при следующем платеже сумма будет 3000 ₽.
